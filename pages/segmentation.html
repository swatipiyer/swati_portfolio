Machine Learning · Team Project

Customer Segmentation

A supervised learning project that classifies customer spending habits by looking at demographic data. We built and tested multiple ML models to find which one predicted best.

Role: Data analysis and model evaluation Tools: Python, scikit-learn, Streamlit and Kaggle Team: Swati Iyer, Kyle Jow, Daphne Loustalet, BaoTran Tran, Joshua Winerman

Overview

Can you predict how someone spends money just from knowing their age, gender, and income? That was the big question. We grabbed a Kaggle dataset with 200 customers and spending scores from 0 to 100, then built and compared multiple models to sort people into Low, Medium, and High spenders.

We went through the usual steps: cleaning up the data, training models, testing them out, and then wrapped it all up in a Streamlit app so people could plug in their numbers and see what category they'd land in.

Data preprocessing Helped categorize raw spending scores into Low (0–33), Medium (34–66), and High (67–100) segments for classification.
Model comparison Evaluated multiple approaches including SVMs, Decision Trees, and Neural Networks to identify the strongest performer.
Visualization Created violin plots, scatterplots, and correlation matrices to explore relationships in the data.

The Core Problem

Businesses want to understand their customers, but raw data doesn't tell you much on its own. We wanted to see if basic demographic info, just gender and age and annual income, could actually predict spending behavior.

The tricky part was working with only 200 rows when most similar Kaggle datasets have 14,000+. We still needed to get results that actually meant something, not just random numbers.

Supervised Learning Classification Neural Networks Data Visualization

Methodology

We followed four main steps: get the data, clean it up and normalize it, train a bunch of models, then test each one using precision, recall, accuracy, and F1-score.

Step 1

Data

Sourced a Kaggle dataset with gender, age, annual income, and spending score (0–100) for 200 customers.

Step 2

Preprocessing

Categorized spending scores into Low/Medium/High buckets and normalized features using Min-Max scaling.

Step 3

Training

Tested Logistic Regression, multiple SVMs (Linear, RBF), Decision Trees, and a Neural Network with 80:20 train/test split.

Step 4

Evaluation

Compared models using precision, recall, accuracy, and F1-score. Tuned the Neural Network with hyperparameter search.

Results

Model Comparison

SVM (Linear Kernel)
Got about 62% accuracy, but it wasn't great with the small dataset and categories that overlapped.
Decision Tree & Logistic Regression
Both did okay, similar to each other, with moderate accuracy on both.
Neural Network
Once we tuned the hyperparameters, it hit 80% accuracy. That was the winner.

Neural Network Architecture

Network structure
4-layer dense network: input → 39 nodes → 45 nodes → 33 nodes → 3 output classes
Hyperparameter tuning
Tested node counts in steps of 3, learning rates of 0.01, 0.001, and 0.0001
Performance
Final model significantly outperformed all other approaches on this dataset.

Discussion

The biggest issue was that we only had 200 data points. Models tend to overfit with that little data, so the results are more like a proof that it can work rather than something you'd actually deploy. But the neural network did pretty well, which shows that with the right setup and tuning, you can still find patterns even in small datasets.

If we did this again, I'd grab a way bigger dataset, try unsupervised methods like K-Means to compare, and spend more time on feature engineering.
