Password Generator

A personalized password generator that uses security questions to create strong, memorable passwords. Built with actual human memory in mind.

Role: Prototyping + Python development Tools: Python, Figma, user interviews, cognitive walkthrough Team: 6-person team (Swati Iyer, Bipasha Sengupta, Anayeli Martinez, Yi-Hsuan Chiang, Xinle Xu, John Missal)

Overview

Most password generators spit out random strings that nobody can remember. We wanted to flip that. Build a generator that creates strong passwords from things you actually know, like your favorite food or the city you grew up in.

The project followed a full HCI design process. Need-finding interviews, personas, prototyping in Figma, cognitive walkthroughs, and a working Python implementation. The result was a command-line tool that asks you to pick 3 of 7 personal security questions, then generates a secure password from your answers.

The Core Problem

There's a tension between security and usability in passwords. Strong passwords are hard to remember. Easy passwords are easy to crack. We started with a simple question. What if the password was built from things you already know, but scrambled so it's still cryptographically strong?

Our interviews with UC Davis students confirmed this tension. People reuse passwords, forget complex ones, and rarely use password managers. The opportunity was to create something personal enough to remember but random enough to be secure.

Human Memory Security Personalization HCI Design

Design Process

Four clear phases. Each one built on the last.

Phase 1

Need Finding

Interviewed 5 UC Davis students about their password habits, frustrations, and what they wished generators did differently.

Phase 2

Personas

Created two personas, including Emerson Jane. A managerial economics student who wants secure but memorable passwords without writing them down.

Phase 3

Prototyping

Built a Figma mobile app prototype and then a Python command-line version. Users pick 3 of 7 security questions and their answers feed into password generation.

Phase 4

Evaluation

Ran cognitive walkthroughs to test clarity of question selection, error handling, and password display. Then iterated based on what we found.

How It Works

Security Questions

Users choose 3 from 7 personal questions. Elementary school name, favorite food, favorite number, parent's birthday, city born in, favorite pastime, and where they see themselves in 10 years. Answers get combined and transformed into the password seed.

Password Generation

The Python function takes user answers, capitalizes and joins them as a base string, then mixes in uppercase and lowercase letters, digits, and symbols. Character requirements are enforced, the final string gets shuffled for security. Users can control whether digits and symbols are included.

HCI Principles

Error Prevention and Recovery

The interface validates inputs, prevents duplicate question selection, and gives clear error messages when users enter invalid choices. Recovery paths are always visible.

User Control

Users maintain control over which questions to answer, whether to include special characters, and can regenerate passwords without restarting. The system adapts to preferences rather than forcing a single approach.

What I Took Away

This project was a good reminder that security tools don't have to feel hostile. The HCI lens forced us to think about the person on the other end, not just the algorithm. The biggest design challenge was balancing personalization, which makes passwords memorable, with randomization, which makes them secure.

If we had more time, I'd combine the Figma frontend with the Python backend into a full web app. Add password strength scoring with real-time feedback. Explore more sophisticated hashing for the answer-to-password pipeline.
