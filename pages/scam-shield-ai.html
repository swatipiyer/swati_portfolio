Scam Shield AI

A web app that detects scams in real time by combining local analysis with AI. Built on the belief that protecting people shouldn't need a subscription or a CS degree.

Role: Product Designer & Frontend Tools: React, Tailwind, Figma, Google Gemini API, Twilio Team: 4-person hackathon team View on Devpost →

Overview

Scam calls and messages targeting elderly people are everywhere. Someone impersonating a police officer, a doctor, a Medicare representative. And the person on the other end often has no one there to tell them it's fake. That was the starting point for this.

We built ScamShield AI to analyze text-based documents, emails, contracts, messages. It flags the suspicious stuff in plain language. Two modes: a local engine that catches common red flags instantly, or optional AI mode using Google Gemini 1.5 Flash for deeper analysis. No account required. Your data stays in your browser unless you choose otherwise.

Why This Problem

Financial scams targeting older adults are getting worse. Phone calls pretending to be from the police or a doctor's office. Emails that look real enough to fool anyone. Financial harm is healthcare. Medical debt, drained savings, the stress of realizing you've been taken advantage of. It's all connected.

Most existing products focus on spam filtering, blocking known bad numbers or flagging junk mail. We were more interested in the calls and messages that get through, the ones that sound legitimate. That's where the real damage happens.

The Approach That Didn't Ship (But Shaped Everything)

Before we built the text analyzer, we spent real time designing a phone call interception system. The idea: incoming calls would route through a Twilio-powered AI agent that listens in, detects scam patterns in real time, and either flags the call or transfers it to a family member.

We mapped out the full architecture: Twilio for telephony, a Node.js webhook server, an NLP-based AI agent for scam detection, and a Redis/Firestore data store for continuous learning. Users would register through a privacy vault (Skyflow), whitelist trusted contacts, and let the system screen everything else.

We dropped it for the hackathon because the scope was too big for a weekend, but the thinking directly shaped our final product. The privacy-first ethos, the hybrid AI approach, the focus on making results understandable instead of just accurate. All of that came from this earlier exploration.

How It Works

Local Analysis

The default mode. A JavaScript engine on your device scans pasted text against a library of regex patterns, scam keywords, and known red flags. Results are instant and nothing leaves your browser.

This mattered to us. Not everyone wants to connect an API key, and the basic protection should work on its own.

AI-Enhanced Mode

Users who want deeper analysis can add their own free Google AI API key. The app sends the text to Gemini 1.5 Flash, which returns a plain-language summary, contextual red flag detection, and a risk score with reasoning.

The key stays in your browser. Never touches our servers. If the API call fails, the app falls back to local analysis automatically.

Design Decisions

Bring Your Own Key

We didn't want to run a server that handles API costs or stores user data. With bring-your-own-key, the app stays free, privacy is airtight, and users stay in control. Google's free tier covers the rest.

Always-Visible Mode State

When you're analyzing text, you should know if you're using local analysis or AI. We added persistent purple "AI Enhanced" badges, clear loading states, and mode indicators throughout so there's no confusion.

Plain English First

The AI output isn't a confidence score in a dashboard. It's a clear explanation of what the text is doing and why it's suspicious. If your grandma can't understand the warning, we failed.

Graceful Fallback

If the Gemini API fails, the app doesn't show an error. It falls back to local analysis silently and gives you results anyway. You always get an answer.

What I'd Do Next

Phone calls came up constantly in our research. Text analysis helps, but the real-time call interception system, the one we designed but couldn't build in a weekend, that's where this needs to go. A browser extension that scans emails and web pages in place would be next, plus OCR so you could photograph a physical letter and analyze it instantly.

The other piece is community patterns. Right now the local engine uses a static library. Opening that up so users can flag new scam types and feed them back would make the baseline protection smarter over time without depending on AI.
